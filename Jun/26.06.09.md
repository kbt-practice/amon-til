### 새로 배운 내용
#### 주제 1: 스케줄러

**문제점**

1. 자원 독점(무거운 연산 하나를 CPU가 계속 처리) 시 다른 작업은 수행하지 못함
2. 우선순위가 밀려 기아 상태 프로세스가 생김
3. CPU가 작업속도가 빨라서 디스크를 기다리느라 CPU Idle이 놀게됨 → 시스템이 낭비됨

**스케줄러 사용 이유**

1. CPU 이용률 최대화 : CPU가 컴퓨터 자원 중 가장 비쌈
2. 처리량을 늘려 총 처리 시간을 감소
3. 대기, 응답 시간 최소화

**기능**

1. 작업 선택 : 어떤 프로세스, 스레드에게 CPU 할당할지 결정
2. 자원 분배 : 타임 슬라이스를 통해 자원을 분배함
3. 컨텍스트 스위칭 관리 : 기존 작업 PCB에 저장 후 새 작업 PCB에서 불러오기

**종류**

1. 장기 스케줄러 : 보조기억장치(가장 느림)에 있는 프로그램 중 어떤 걸 메모리(RAM)에 올릴지
2. 중기 스케줄러 : 메모리 부족 시 하나의 프로세스(가장 오래 안 쓴 + 상태 + 우선순위 + 크기 고려)를 디스크의 스왑 공간으로 옮겨버림
3. 단기 스케줄러 : CPU에서 준비 상태로 있는 프로세스 중 CPU를 할당받을 프로세스 결정

**스케줄링**

동시에, 병렬적으로 어떤 걸 언제, 얼만큼 실행할까? → 규칙, 정책을 만들어 놓은것

**레벨**

1. 고수준 스케줄링 : 메모리에 올릴 프로세스 결정 → 시스템의 전체 부하 조절
2. 중간 스케줄링 : 메모리 부족 시 일부 프로세스를 잠시 내보냄
3. 저수준 스케줄링 : CPU를 누구에게 할당할지
    
```
Job Queue (작업 대기 큐)

-> 이 중 누구를 올리자 (고수준 스케줄링)

Ready Queue (실행 준비 작업들의 대기 줄)

-> 프로세스 선택 

-> CPU 할당 (저수준 스케줄링)
```

스케줄러는 ~일을 하는 주체이고 스케줄링은 ~일의 방법

**알고리즘**

FSFS : 먼저 온 사람 먼저 → 구현 쉬움 / 많이 기다릴 수도

SJF : 가장 짧은 작업 먼저 → 평균 대기시간 적음 / 기아상태 가능성

SRTF : 남은 시간 가장 짧은 거 먼저. SJF 선점형 버전(중간에 바뀔 수 있음) → 응답성 좋음 / 컨텍스트 스위칭 잦음

Priority : 우선순위 높은 거 먼저 → 중요한 작업 우선 수행 / 기아상태 가능성

Round Robin : 모든 프로세스에게 동일한 시간 제공 → 공정함 / 시간에 따른 성능 차이

Multilevel Queue : 프로세스를 그룹별로 나누어 다른 알고리즘을 사용해 관리

현업에서 스케줄링 사용

1. 백엔드 개발자 : 사용 안함. 운영체제에 결정되어 있음. 성능 최적화 트러블 슈팅 시 사용.
2. 인프라 엔지니어 : 일부 선택 가능

#### 주제 2: 병렬성

여러 CPU 또는 프로세서를 사용해 여러 작업 동시 수행

왜 사용? : 효율적인 자원 사용 → 실시간, 대용량, CPU 활용도 증가, 성능 향상

병렬성은 동시성의 한 형태임

병렬 동작 방식 : 작업 분할 → 코어 할당 → 병렬 실행 → 결과 통합 → 오류 처리

주의 : 병렬 분할/병합 과정 비쌈, 공유 자원 접근 시 주의, 물리적인 제한이 있을 수 있음(하드웨어 정보 확인)
