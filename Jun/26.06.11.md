### 새로 배운 내용
#### 주제 1: 교착상태(deadlock)
두 개 이상의 프로세스나 쓰레드가 자원을 점유한 채 서로가 필요로 하는 자원을 기다리며 무한히 대기하는 상태

알아야 하는 이유 : 시스템 안정성 확보, 자원 효율성 확보, 장애 원인 분석, 동시성 코드 설계. 가장 큰 이유는 에러가 바로 티가 나지 않음

**실무**

- DB 트랜잭션 : 트랜잭션 테스트 진행
- 멀티스레드 코드 : 코드 레벨에서 생기는 문제
- 커넥션 풀 : spike traffic / 갑자기 요청이 많아져서 DB가 죽고 서비스가 죽게됨
    - 커넥션 풀은 WAS와 DB에 있는데, DB의 커넥션 풀이 더 중요함
- 파일 처리 : 거의 없음

**교착 상태 조건** - 아래 4가지 모두 만족해야만 함

1. 상호 배제 : 하나의 자원을 한 번에 하나의 프로세스나 스레드에서만 사용
2. 점유와 대기 : 다른 자원을 기다림 → 줄이기
    1. 필요한 자원 한번에 얻기 또는 실패 시 기존 자원 놓기(타임아웃, 락 범위 최소화)
3. 비선점 : 다른 프로세스가 점유한 자원을 빼앗을 수 없음
4. 순환 대기 : 자원을 기다리는 관계가 원형 구조를 이룸 → 규칙화
    1. 모든 스레드가 자원을 같은 순서로 획득하도록 규칙화

#### 주제 2: 기아상태
프로세스가 계속 실행되지 못하는 것

**문제 발생 상황**

1. 우선 순위 기반 스케쥴링 : 우선순위가 낮으면 기아상태가 됨
2. 불공정한 락 획득 방식일 시 : 특정 스레드가 반복적으로 락 획득 실패
    1. 공정락은 계속 관리를 해줘야해서 자원이 소모됨
    2. 분공정락은 훨씬 저렴하게 운영 가능 → 디폴트
3. Read-Write 에서 Write 기아
    1. Reader는 계속 읽기 가능 → Reader가 끊이질 않ㅇ므 → Writer 기아 상태

**해결 방법**

1. 에이징 : 오래된 작업의 우선순위를 높임
2. 라운드 로빈 : 모든 작업한테 일정한 시간을 할당하고, 그 시간만큼 수행하게 함
3. 최대 대기 시간 설정 : 일정 시간 이상 기다린 작업은 강제로 실행 시도
4. 공정한 큐 사용 : 누가 먼저 기다렸는지 구분 (FIFO, Fair Lock, 대기열 기반 처리)

기아상태는 보통 운영체제가 아닌 곳에서 발생 : 스레드(Spring) 풀, 커넥션(DB) 풀, 락, 큐 …

실무 - **DB 커넥션 풀 기아 상태**

1. 디도스 : DB 고갈, 웹서버 고갈
2. 커넥션 타임아웃, 맥스 풀 사이즈, 슬로우 쿼리(오랜 기간 점유 쿼리) 제거, 트랜잭션 범위 축소

**모든 에러 대응의 대원칙**

1. 애플리케이션, DB, JVM, 쿠버네티스, 메세지 큐 관측
2. 에러 발생 안하게 한다 → 힘듦… 빠르게 발견하고 고치는 게 좋음

#### 주제 3: 세마포어
세마포어로 동시 접근 수 제한 → 자원 보호, 동시성 제어, 시스템 안정성

**특징**

1. 세마포어 값 S : 사용 가능한 남은 허가권의 개수
2. P 연산 : 허가권 하나 획득 시 S를 1 감소 → 더이상 허가권 없으면 대기
3. V 연산 : 자원 사용 완료 시 S를 1 증가 → 대기 중인 스레드 하나가 깨어남

**종류**

1. 이진 세마포어
    1. 0, 1만 가지는 세마포어
    2. 뮤텍스는 소유권 개념이 있지만 세마포어는 몇개를 허가해주냐의 차이
2. 카운팅 세마포어 → 일반적
    1. 값이 0이상인 정수 → 여러개의 자원 접근 제어
    2. 커넥션 풀, 스레드 풀, 여러대의 프린트 등
    3. 동시 접근 가능한 작업 수 제어

**사용 사례**

1. 외부 API 호출 제한
    1. 외부 API 연동 된 곳으로 과다 요청 → 외부 API 가 서버 자체 IP를 막으면서 문제가 생길 수 있음
    2. 외부 API가 중요한 자원이라면 큰 문제
2. 파일, 이미지 처리 서버 : 메모리 자원이 상당히 많이 들어 다른 작업에 영향이 갈 수 있음
3. 배치 작업
    1. 한번에 처리할 때 자원이 많이 필요한 경우
    2. 이미지 삭제, 이미지 포맷 변환, 이미지 해상도 다변화 …
4. 로드밸런서 : DB 작업을 배분할 때 동시 요청 수 제한
5. 엔진엑스 웹서버 동시 연결 제한 : HTTP 요청 개수 제한
