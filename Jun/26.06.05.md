### 새로 배운 내용
#### 주제 1: JPA 키 생성 전략이 Entity 저장 시점, INSERT 실행 방식, 대량 데이터 처리 성능에 미치는 영향을 분석하시오.
JPA는 영속성 컨텍스트를 통해 쓰기 지연과 Batch를 지원하지만, 키 생성 전략이 이 흐름을 근본적으로 바꾼다.

IDENTITY는 INSERT 후 DB가 PK를 돌려주는 구조라, persist() 시점에 즉시 INSERT를 실행해야 한다. 이렇게 되면 쓰기 지연이 깨지고 Hibernate는 Batch를 구성할 수 없어, 10만 건 저장 = 10만 번 DB 왕복이 된다.

SEQUENCE는 allocationSize로 PK를 미리 선점하므로 INSERT 전 PK가 확정된다. 쓰기 지연이 유지되고 ActionQueue에 INSERT를 쌓은 뒤 flush 시 Batch로 일괄 전송된다. 여기에 batch_size와 MySQL 이라면 rewriteBatchedStatements를 더해 패킷을 조절하여 성능을 더 높인다.

결국 대량 저장이 필요한 MySQL 환경이라면 (SEQUENCE + batch_size)를 가장한 JDBC Batch + rewriteBatchedStatements 조합이 필수이다. IDENTITY 그대로 saveAll()을 쓰는 건 단건 INSERT를 반복해주는 함수이기 때문에 지양하는 것이 좋다.
