### 새로 배운 내용
#### 주제 1: git
**기본 명령어**
- git init : Working Directory 생성 (Unstaged Files, Untracked Files)
- git add : stage로 이동
- 파일 변경 시 : WD의 Unstaged Files로 이동
- git commit : History로 이동
- git remote : 로컬에 저장된 원격 저장소 별칭 지정 (기본 origin)
- git pull = git fetch + git merge
- git stash : 임시 저장, 최근 커밋과의 차이점 보관

**백업**
- git restore : 커밋하지 않은 변경사항을 마지막 상태로 되돌림 → 파일 단위 관리
- git revert : 되돌아가는데 되돌아간다는 기록이 남아있음
- git reset : 작업하던 곳을 이전 상태로 되돌리기 → 브랜치, 커밋 단위 관리

Q. git clone 하면 git remote 가 실행되는 것인가?
A. 맞음. 초기화 → remote → 모든 원격 정보 다운 → WD 체크아웃

#### 주제 2: 객체지향언어
- 클래스 : 속성과 함수를 가지고 있는 상자라고 생각하면 됨
- 매서드 : 클래스의 함수를 의미함
- 생성자 : 클래스의 초기화를 담당함
- 객체 : 속성을 가지고 있고 함수를 수행하는 것
- 상속 : 상위 클래스에 있던 속성과 매서드를 하위 클래스에 위임하는 것
