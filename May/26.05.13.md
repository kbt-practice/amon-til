- 자바

    CPU 마다 번역이 달랐음

    C, C++ 은 운영체제를 만들기 위해 만들어졌기 때문에 어려워짐

    C언어는 기계어에 가까워 3가지 문제가 존재

    1. 컴파일러가 달라지면 결과물이 달라짐(윈도우, 유닉스 등)
    2. 하드웨어 제어 방식이 다름(삼성, 소니 등)
    3. 소스코드 복잡성 : 절차지향이라 코드가 복잡해지면 유지보수가 힘들고 조건문이 많아짐

    → Porting Hell(제조사별로 다르게 준비)

    → CPU에 상관없이 어디서든 돌아가는 언어를 만들자! 바이트코드인 JVM을 포함한 자바 탄생

    **자바의 특징**

    1. 플랫폼 독립성 (JVM) : 바이트 코드로 어디서든 실행함
    2. 보안 안정성 : 운영체제 위에서 바이트 코드를 기계어로 변환
    3. 메모리 관리 : 메모리 직접 관리 시 관리가 어려워 자동 메모리 관리함
    4. 객체 지향언어

    JDK ⊃ JRE(가장 기본적인 프로그램 베이스) ⊃ JVM

    - JVM 단독 실행 불가 / JRE 단독 실행 가능
    - JVM : 자바의 바이트 코드를 실행하는 (기계어로 실행하는) 엔진
    - JRE : 자바 프로그램을 실행하기 위한 **최소한**의 환경
    - JDK : 바이트 코드를 만들기 위한 도구. JRE + 개발용 도구

    C언어는 malloc, free를 통해 개발자가 메모리를 관리함

    - 속도가 빠르고 메모리 가성비가 좋음
    - 메모리 누수나 잘못된 접근을 할 수 있음

    자바는 개발자가 관리하지 않음

    - JVM을 실행 → 운영체에제에 메모리 할당 → 메모리 할당
    - 스택 : 함수 안에서 관리. 자동 할당 및 해제
    - 힙 : 모든 실행 흐름을 한 곳에서 보는 메모리 공간. 공용 사용 공간
    - GC (Garbage Collector) : 스택 기반
        - 더이상 참조되지 않는 힙 영역 메모리 제거
        - 어떠한 변수도 힙에 있는 객체를 가리키고 있지 않을 때 청소

    자바는 정적으로 타입을 지정해줘야 함

    추정하는 키워드 var는 지역 변수에서만 사용 가능

    스택에는 기본 타입 힙엔 참조 타입

    C언어의 포인터는 실제 하드웨어 메모리의 참조값이고 참조 타입의 참조는 상대적인 메모리의 참조값

    String의 내용이 변경될 때는 새로 메모리를 할당해서 객체를 만들고 참조를 변경하는 것임. 기존 데이터는 따로 저장되어 나중에 똑같은 String의 내용이 사용될 때 참조되고 오랜 기간 참조가 되지 않으면 GC가 정리해버림.

    new 키워드를 사용하면 런타임 중 할당됨

    출력 : System.out.println("이름: " + name);

- 한 줄 정리 키워드
    - 컴퓨터언어 : 사람이 명령을 내리고 기계가 명령을 수행할 수 있게 돕는 의사소통 도구이다. 예를 들어 java, c++ 등이 있다.
    - 자바 : 자바는 다양한 환경에서도 똑같은 결과물을 도출하는 컴퓨터언어이다. 자동으로 메모리를 관리해주며, 객체끼리 상호작용하며 동작하는 객체지향언어이다.
    - JDK : 컴퓨터언어를 기계어로 변경해주는 도구이다. 코드가 실행될 환경을 만들고, 코드를 컴파일, 디버깅, 문서화, 패키징 한다. 네가지 단계가 없다면 코드는 존재하지만 실행은 할 수 없다.
    - JRE : 자바를 사용하여 애플리케이션을 만들기 위해 필요한 도구들이다. 다양한 환경에서도 동일한 환경으로 만들어주는 가상 머신인 JVM과 기본적인 개발을 위한 핵심 라이브러리인 Java Class Library로 구성되어 있다.
    - 바이트코드 : 자바로 작성된 코드를 컴파일 하면 생기는 바이트로 구성된 데이터의 집합이다. 사람과 운영체제 모두 읽을 수 없다.
    - GC : 메모리에 더이상 참조되지 않는 버려진 객체를 회수하는 것을 의미하며 힙이 어느정도 찼을 때 한번씩 실행된다. 정리한 메모리 내에서는 단편화가 이루어지지 않도록 한곳으로 정리한다.
    - 스택 : 함수 안에서 할당되고 해지되며 바로 사용되는 지역변수, 매개변수가 저장된다.
    - 힙 : 개발자가 동적으로 할당되는 메모리 영역이다. 자바는 GC가 동적 할당을 담당한다.

한 일 : 기본적인 자바 환경과 구성에 대해 배웠다.

부족한 점 : 스택과 힙 부분에서 자료구조와 운영체제 메모리 부분에서 헷갈렸어서 개념을 다시 정리했다.
